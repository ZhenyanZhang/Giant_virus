# Workflow for Machine Learning

## Step 1: Data preparation and feature selection
In this step, we collected the **environmental factors** online (details can be found in the manuscript) and curated the metadata to extract three critical **technical covariates** for each sample.
1. Downloaded the data from [Climate Data Store](https://cds.climate.copernicus.eu/datasets/reanalysis-era5-single-levels-monthly-means?tab=overview), [HydroLAKES database](https://www.hydrosheds.org/products/hydrolakes) and [previous study](https://www.nature.com/articles/s41597-022-01284-8); three critical technical covariates for each sample were extracted from the metadata of each independent study.
2. To prevent multicollinearity in the machine-earning models, we evaluated Spearman’s rank correlation among all **environmental factors** using [spearman_cor.R](spearman_cor.R). For highly correlated factors (|Spearman’s _r_| > 0.6), we exclusively retained the most representative and biologically meaningful one.

## Step 2: Machine-learning model construction and validation
We identified the most suitable algorithm and associated hyperparameters for machine learning by first constructing regression models using different algorithms (XGBoost and random forest) with different combinations of hyperparameters. After hyperparametric tuning, the best combination of hyperparameters for each algorithm and each microbial index was evaluated using the performance of the models (represented as out-of-fold R2, RMSE and MAE). To evaluate the ability of extrapolation of machine-learning models, grouped cross-validation (which grouped by sampling sites) and spatial cross-validation were also used. All these work were performed by [Machine_learning_cv.py](Machine_learning_cv.py).

## Step 3: Environmental outliers removal from predicting dataset
We calculated the Mahalanobis distance within a 9-dimensional environmental space defined by the predictors used for modeling. This metric quantified the multivariate distance of any target lake to the statistical centroid (multivariate mean) of the training dataset. Under the assumption of multivariate normality, the squared Mahalanobis distance theoretically follows a chi-squared distribution with degrees of freedom equal to the number of predictor dimensions. Accordingly, we defined our reliable prediction boundary and removed the environmental outliers, which were estimated for a chi-squared value of 0.9. Because environmental conditions in global lakes fluctuated from 2000 to 2022, we evaluated this outlier status monthly. Ultimately, lakes identified as outliers in more than 50% of the evaluated months were strictly excluded from downstream predictions. In addition to spatial filtering, we implemented a strict temporal constraint. Because the majority of samples in our dataset were collected between July and September, we restricted our downstream predictions exclusively to these months. This step was finished by [environmental_outliers_identify.py](environmental_outliers_identify.py) and [filter_lake.py](filter_lake.py).

## Step 4: Predicting
We performed 99 independent bootstrap resampling iterations (sampling with replacement) on the training dataset. For each iteration, a distinct XGBoost model was trained with the best combinations of hyperparameters defined by random cross-validation. The final predicted value for each lake was determined as the ensemble mean of the 99 individual predictions. Compared to single-model predictions, aggregating these iterations significantly reduced overall model variance. Finally, to quantify the prediction uncertainty of each lake, we calculated the interquartile range (IQR) across the 99 predictions, where a narrower IQR indicates higher model consensus and predictive reliability. All the above steps were performed using [Bootstrap_predict.py](Bootstrap_predict.py). After that, the annual dynamics of richness and dissimilarity were determined using linear regression, which was performed using [annual_trend.py](annual_trend.py).
